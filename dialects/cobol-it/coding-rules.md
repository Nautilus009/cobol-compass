# COBOL-IT Coding Rules

- Confirm whether the source is fixed format or free format.
- Preserve existing period usage where possible.
- Check compiler behavior before assuming IBM Enterprise COBOL behavior.
- Pay special attention to file status behavior, EXTFH behavior, sort behavior, and runtime configuration.
- Validate generated code with the COBOL-IT compiler before use.
- Preserve existing copybook inclusion style.
- Avoid introducing dialect features unless the project already uses them or the active profile approves them.
