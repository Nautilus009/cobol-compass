# COBOL Development Principles

- Preserve existing business behavior.
- Prefer small, reviewable changes.
- Understand the dialect before changing syntax.
- Understand the runtime before changing file, database, transaction, or messaging logic.
- Do not assume all COBOL dialects behave the same.
- Treat generated code as a draft until reviewed and tested.
- Explain risk clearly.
- Always provide a focused test plan for any proposed change.
- Preserve data layouts unless the requested change explicitly requires changing them.
- Review numeric precision, signed fields, redefines, occurs clauses, and copybook impacts carefully.
