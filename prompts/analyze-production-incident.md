# Analyze Production Incident

Analyze the provided COBOL code, logs, runtime context, and incident notes.

Use the active Cobol Compass profile.

## Required Output

1. Active profile assumptions
2. Observed facts
3. Timeline
4. Suspected failure point
5. Likely causes
6. Unknowns
7. Evidence to collect next
8. Minimal mitigation options
9. Risk level
10. Follow-up test plan

## Rules

- Separate observed facts from assumptions.
- Do not invent copybooks, file formats, message formats, or database schemas.
- Do not propose code changes until the failure mode is reasonably understood.
