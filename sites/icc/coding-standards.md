# ICC COBOL-IT Coding Standards

ICC is a conservative production environment.

Rules:

- Prefer the simplest, lowest-risk change.
- Do not rewrite unrelated code.
- Use spaces, not tabs.
- Do not use COBOL inline comments.
- COBOL comment lines must start with an asterisk in column 1.
- Every COBOL paragraph must end with a dot.
- Preserve existing period usage on untouched lines.
- Follow nearby code examples.
- Follow existing status, free-text, DISPLAY, and rollback patterns.
- In ICC C integration, remember that C long variables are 64-bit by default.
