# ICC Minimal Change Policy

For ICC work, the preferred fix is the smallest clear change that solves the requested issue and preserves existing production behavior.

Avoid:

- Broad formatting changes
- Paragraph reorganization
- New helper abstractions without a clear local pattern
- Copybook layout changes unless required
- Changes to unrelated DISPLAY text, status fields, or return-code paths

Every change proposal should include a short risk summary and focused test plan.
